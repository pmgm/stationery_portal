Purpose and Conventions

Purpose and Conventions

Purpose

Cgiapp2 is a PHP5 abstract class; it is not intended that this class be instantiated directly. Instead, it is intended that your application will be implemented as a subclass (aka extension) of Cgiapp2.

To inherit from Cgiapp2, simply create a new class that extends it:

  1. require_once 'Cgiapp2.class.php';
    2. 

  2. class WebApp extends Cgiapp2 { ... }
    3.

Notation and Conventions

For the purpose of this document, we will refer to the following conventions:

  • Application Class: A Cgiapp2 subclass/extension.
  • Instance Script: A PHP script that creates an object instance of the Application Class.
  • WebApp.class.php: The PHP file which implements your Application Class.
  • WebApp: The actual Application Class.
  • webapp.php: The Instance Script which implements your Application Class.
  • $webapp: An instance (object) of your Application Class.
  • $this: Same as $webapp, used in instance methods to pass around the current object. (Standard PHP Object-Oriented syntax)

IMPORTANT NOTE ABOUT RUN MODE METHODS

In the perl class from which this is derived, you would never send output to the screen from your application; you would leave this up to the run() method, which would load your template(s) and send the entire page at once. Part of the reason for this is that in perl, you need to send your HTTP headers manually each time, and the run() method takes care of that for you.

This is not the case with PHP, though I would advise to try to follow this methodology as closely as possible. Breaking this rule can be a source of errors, particularly if you are trying to set any non-standard HTTP headers using Cgiapp2's various header methods, or if you ar setting cookies.

One reason for using this class is because you want to separate program logic from design logic. This may be especially important if the design team works separately from the programming team, or if you are trying to make applications portable. For these reasons alone, you should attempt to only create output using the run method, which will fill templates according to the run mode called.
