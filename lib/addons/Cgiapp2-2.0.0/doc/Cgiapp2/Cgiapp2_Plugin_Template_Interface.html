Cgiapp2_Plugin_Template_Interface

Defines an interface for a template plugin. A template engine should  implement an initialization method,  a variable assignment method, and  a method for fetching the compiled template.
Template plugins that utilize this interface will work seamlessly with  Cgiapp2's Cgiapp2::tmpl_path(), Cgiapp2::tmpl_assign(), and  Cgiapp2::load_tmpl() methods.
Since a Cgiapp2 derived class is passed to each, the template engine  may choose to register itself with that class using the  Cgiapp2::param() method. Alternatively, it may implement a singleton;  these details are left to the programmer.
The plugin class should then register itself with Cgiapp2 or a Cgiapp2-derived  class via Cgiapp2::add_callback():  
  1.  class MyTemplatePlugin implements Cgiapp2_Plugin_Template_Interface { //... }
    2. 

  2.     Cgiapp2::add_callback('tmpl_path', array('MyTemplatePlugin', 'init'), 'Cgiapp2');
    3. 

  3.   Cgiapp2::add_callback('tmpl_assign', array('MyTemplatePlugin', 'assign'), 'Cgiapp2');
    4. 

  4.   Cgiapp2::add_callback('tmpl_fetch', array('MyTemplatePlugin', 'fetch'), 'Cgiapp2');
    5.
