Smarty Template Plugin for Cgiapp2

Implements Cgiapp2_Plugin_Template_Interface to create a Smarty plugin  for Cgiapp2.
Additionally, Cgiapp2_Plugin_Smarty implements a singleton and dynamic proxy.  This allows you to do the following:  
  1.  // Register rawurlencode() with Smarty engine.
    2. 

  2.   Cgiapp2_Plugin_Smarty::getInstance()->register_modifier('rawurlencode', 'rawurlencode');
    3. 

Registers with Cgiapp2's tmpl_path, tmpl_assign, and tmpl_fetch hooks;  registration is done with the Cgiapp2 class.

method init [line 257]

	

    

    

		bool init(
Cgiapp2
$cgiapp, mixed
$tmpl_path, [mixed
$extra_params = null])
    

    


	
		Initialize a template instance and/or set the template path

If no Cgiapp2_Plugin_Smarty instance currently exists, it is first  initialized via getInstance() using the template path, extra  parameters, and Cgiapp2-based object instance. Smarty is looked for in the  Cgiapp2 instance parameter 'SMARTY_PATH'; if not found, or an error occurs  initializing Smarty, a warning is raised.
init() is used by Smarty to set the template root path to each of the  required Smarty template directories (templates, templates_c, configs,  and cache).  If the template path has not changed, nothing is done.
