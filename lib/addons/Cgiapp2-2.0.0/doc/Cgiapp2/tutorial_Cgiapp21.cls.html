Usage Example

A simple form -> list -> show detail application

Imagine you have to write an application to search through a database of widgets. Your application has three screens:

  • Search form
  • List of results
  • Detail of a single record

To write this application using Cgiapp2 you will create two files:

  • WidgetView.class.php -- Your "Application Class"
  • widgetview.php -- Your "Instance Script"

The Application Class contains all the code specific to your application functionality, and it exists outside of your web server's document root, somewhere in the PHP include path.

The Instance Script is what is actually called by your web server. It is a very small, simple file which simply creates an instance of your application and calls an inherited method, run(). Following is the entirety of "widgetview.php":

  1. <?php
    2. 

  2.    require_once 'WidgetView.class.php';
    3. 

  3.    $webapp = new WidgetView();
    4. 

  4.    $webapp->run();
    5. 

  5. ?>
    6.

Cgiapp2 takes care of implementing the run() method. Notice that at no point do you call echo() to send any output to the browser. Instead, all output is returned as a scalar.

Cgiapp2's most significant contribution is in managing the application state. Notice that all which is needed to push the application forward is to set the value of a HTML form parameter 'rm' to the value of the "run mode" you wish to handle the form submission. This is the key to Cgiapp2.
